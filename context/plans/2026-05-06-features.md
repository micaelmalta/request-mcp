# fetch-mcp Feature Expansion — Master Plan

**Created:** 2026-05-06  
**Scope:** Four features across phases: caching, pdf_fetch, auth headers, test coverage

---

## Core Principles

1. **No new mandatory deps** — every new feature either uses stdlib or makes the new dep optional (lazy import + graceful error message).
2. **Caller transparency** — caching and auth must not silently change observable behaviour; cache hits are logged via `_log_savings`, auth errors surface as `Error: ...` strings.
3. **Backward compatibility** — all new parameters have safe defaults; existing callers see no change.
4. **Test first** — each phase begins by writing tests that fail, then adds the implementation.
5. **One concern per module** — `cache.py`, `pdf.py` as new modules; existing modules are minimally touched.
6. **Savings log continuity** — every tool that returns content must call `_log_savings` regardless of path (cache hit or miss).

---

## Architecture Decisions

| Decision | Choice | Rationale | Alternatives Rejected |
|---|---|---|---|
| Cache backend | In-process `dict` + `time` (stdlib) | Zero deps, fits single-process MCP server lifetime | Redis (overkill, external dep), SQLite (adds complexity for restarts we don't care about) |
| Cache key | SHA-256(`url` + sorted(`headers`)) | Stable, collision-resistant, handles auth header variance | Plain URL string (breaks when same URL used with different auth) |
| PDF extraction | `pdfminer.six` (lazy import) | Pure Python, no native binaries, optional dep | pymupdf (binary dep, adds 15 MB), auto-detect (premature) |
| Auth interface | `headers: dict[str, str] \| None` on `smart_fetch` and `browser_fetch` | Simple, explicit, no secret storage | Env profiles (more complex, deferred), per-tool credential objects (over-engineered) |
| Test approach | `pytest-httpx` for HTTP mocking | Integrates with existing pytest suite, no subprocess needed | `responses` lib (not async-aware), live network (flaky) |

## Graceful Degradation

| Failure Scenario | Expected Behavior |
|---|---|
| Cache module import fails | Never happens — stdlib only |
| `pdfminer.six` not installed | `pdf_fetch` returns `"Error: pdfminer.six is not installed. Run \`uv add pdfminer.six\`."` |
| PDF URL returns non-PDF content | Detected via `Content-Type`; falls back to `smart_fetch` HTML path |
| PDF is scanned (no text layer) | Returns `"Warning: No extractable text found in PDF."` with page count |
| Auth headers contain invalid value | Propagated as HTTP error from the remote server, formatted by `_handle_error` |
| Cache TTL expires mid-session | Next call fetches fresh; no staleness visible to caller |

---

## Progressive Regression Rule

- **After Phase 1:** `tests/test_json_optimizer.py` + all new tests green. No implementation changes, so no regressions possible.
- **After Phase 2:** All Phase 1 tests still green + new cache tests green.
- **After Phase 3:** All prior tests + new auth tests green.
- **After Phase 4:** All prior tests + new pdf_fetch tests green.
